# Sanity Setup Automation

Once you provide your Project ID, I will:

## 1. Update Environment Variables
```bash
# I'll update .env.local with:
NEXT_PUBLIC_SANITY_PROJECT_ID=your-actual-project-id
NEXT_PUBLIC_SANITY_DATASET=production
```

## 2. Create Sanity Schemas
I'll create schema files for:
- Courses
- Testimonials  
- Blog Posts
- Events
- Faculty
- Hero Content

## 3. Test Connection
- Start dev server
- Visit /cms-demo page
- Verify connection status

## 4. Set Up Studio
- Create sanity.config.ts
- Configure studio with your schemas
- Provide studio URL for content management

## 5. Integration Examples
- Show you how to replace static content with CMS data
- Demonstrate live content updates

Just provide your Project ID and I'll handle everything else automatically!
